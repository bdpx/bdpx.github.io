XML схемы

Альтернативу определениям DTD представляют XML схемы. Схемы используют синтаксис языка XML для определения правил словаря XML документа - и это далеко не одно их преимущество, по сравнению с DTD. К достоинствам схем можно отнести так же расширяемость и возможность наследования, поддержку пространств имен, поддержку бóльшего, по сравнению с DTD, количества типов данных (неоспоримое преимущество, если речь идет о программировании, а не разметке) и др.

На данный момент можно выделить 2 основных типа схем: XML-Data Reduced и XML-Schema.

Схема представляет собой XML файл, содержащий описание всех элементов и атрибутов проектируемого словаря XML документа. Ниже приведен фрагмент файла схемы XML-Data Reduced:

<?xml version="1.0" ?>
<Schema xmlns="urn:schemas-microsoft-com:xml-data" xmlns:dt="urn:schemas-microsoft-com:datatypes">
    ...
</Schema>

C документом схема может быть ассоциированна путем декларации пространства имен в корневом элементе XML документа: 

<?xml version="1.0" ?>
<catalog xmlns="x-schema:Schema.xml" >
    ...

Пространства имен представляют собой расширенный синтаксис записи имен XML, позволяющий разрешить конфликт имен, при необходимости использования правил нескольких словарей, определяемых соответствующими схемами, в одном документе. Если в документе используются элементы или атрибуты нескольких словарей, то выйти из положения можно следующим образом: 

<?xml version="1.0" ?>
<catalog xmlns:a="x-schema:schema_a.xml" xmlns:b="x-schema:schema_b.xml">
    <a:some_elt a:attr="value" >
        ...
    <a:some_elt>
    <b:other_elt b:attr="value" />
    ...

В данном примере элементы и атрибуты записанны с использованием расширенного синтаксиса имен XML: 

префикс_пространства_имен:имя_элемента_или_атрибута

а сам префикс пространства имен ссоциируется со схемой в описании корневого элемента (подсвечены в примере выше зеленым фоном): 

xmlns:префикс_пространства_имен="схема"
