XML-Data Reduced

Schema

Элемент Schema языка XML-DR выполняет следующие функции:

  • Содержит декларации элементов и атрибутов
  • Дает схеме имя
  • Декларирует используемые в схеме пространства имен

В каждой схеме XML-DR необходимо объявить пространство имен типов данных Microsoft и XML-Data. Пример объявления схемы приводится ниже:

<Schema name="SchemaExample.xml"
    xmlns="urn:schemas-microsoft-com:xml-data"
    xmlns:dt="urn:schemas-microsoft-com:datatypes">
    <!-- Содержание схемы -->
</Schema>

Элементы

Элементы объявляются в элементах ElementType. Ниже приводится пример объявления элемента: 

Объявление в схеме
<ElementType name="book">
    ...
</ElementType>
 
Аналог в DTD
<!ELEMENT book ...>

Элемент ElementType может содержать пять важных атрибутов:

Атрибут

Значение

name

Имя элемента

content

Описание содержания элемента:

§        empty - пустой

§        textOnly - только текст (#PCDATA)

§        eltOnly - только элементы

§        mixed - смешанное содержание (#PCDATA и элементы)

dt:type

Отмечает тип элемента. Допустимые значения данного атрибута рассматриваются далее.

model

Открытая (open) или закрытая (close) модель содержания.

order

Базовый порядок дочерних элементов:

§        one - один из списка элементов

§        seq - указанная последовательность

§        many - указанные элементы могут присутствовать или не присутствовать, причем в любом порядке.

Содержание элемента описывается при помощи элементов <element > и <attribute >. Элемент <element > задает дочерние элементы описываемого элемента. Он может содержать три атрибута:

Атрибут

Значение

type

Соответствует значению атрибута name элемента elementType, объявленного в схеме

minOccurs

Минимальное количество появлений элемента в данном элементе. Принимаемые значения:

§        0 - элемент необязателен

§        1 - (по умолчанию) элемент должен присутствовать минимум один раз

maxOccurs

Максимальное количество появлений элемента в данном элементе. Принимаемые значения:

§        1 - (по умолчанию) элемент должен присутствовать не более одного раза

§        * - число вхождений элемента не ограниченно

Пример: объявление элемента elt1 со смешанным содержанием (текст и/или elt2) 

Объявление в схеме
<ElementType name="elt1" content="mixed" order="many">
    <element type="elt2" />
<ElementType>
 
Аналог в DTD
<!ELEMENT elt1 (#PCDATA|elt2)* >

Элемент <attribute > так же содержит три атрибута

Атрибут

Значение

type

соответствует значению атрибута name элемента <AttributeType > (см. далее в уроке), определенному в этой схеме

default

Значение атрибута по умолчанию. Переопределяет любое значение по умолчанию, заданное в элементе <AttributeType >, на который он ссылается.

required

Показывает, является ли атрибут обязательным. Принимает значение "yes", если да. Не требуется, если определен в элементе <AttributeType >

и служит для указания атрибутов описываемого элемента.

Пример: описание элемента с пустым содержимым, имеющего обязательный атрибут id типа ID:

Объявление в схеме
<ElementType name="elt1" content="empty" >
    <attribute type="id" required="yes" dt:type="id" />
<ElementType>
 
Аналог в DTD
<!ATTLIST elt1
    id ID #REQUIRED>

Для описания сложных моделей содержания часто применяют элемент <group >, выполняющий роль круглых скобок "(" и ")" в DTD. Вот его атрибуты:

Атрибут

Значение

minOccurs

Минимальное количество появлений элемента в данном элементе. Принимаемые значения:

§        0 - элемент необязателен

§        1 - (по умолчанию) элемент должен присутствовать минимум один раз

maxOccurs

Максимальное количество появлений элемента в данном элементе. Принимаемые значения:

§        1 - (по умолчанию) элемент должен присутствовать не более одного раза

§        * - число вхождений элемента не ограниченно

order

Базовый порядок дочерних элементов:

§        one - один из списка элементов

§        seq - указанная последовательность

§        many - указанные элементы могут присутствовать или не присутствовать, причем в любом порядке.

Атрибуты

Описание атрибутов осуществляется при помощи элемента <AttributeType > 

<AttributeType name="title" required="yes"/>

Элемент может иметь пять атрибутов:

Атрибут

Значение

name

Обязательный атрибут, указывающий имя описываемого атрибута

default

Значение атрибута по умолчанию, должно соответствовать типу, заданному атрибутом dt:type

dt:type

Тип данных описываемого атрибута

dt:value

Список возможных значений (если тип перечислимый)

required

Указывает, является ли атрибут обязательным ("yes" - да, "no" - нет)

Типы данных

В целях облегчения понимания схем, мы не будем глубоко вдаваться в типизацию данных, описываемых при помощи XML-DR. Приведем лишь перечени основных базовых типов данных:

Атрибут

Значение

id, idref, idrefs

Идентификатор, ссылка на id, несколько разделенных пробелами ссылок на id (см. DTD)

entity, entityes

Имя сущности, несколько имен сущностей, разделенных пробелами (см. DTD)

nmtoken, nmtokens

Имя, последовательность имен, разделенных пробклами (см. DTD)

enumeration

Перечисление

string

Строка символов

int

Целочисленный тип

number

Число с плавающей точкой (разрешается запись в экспоненциальной форме)

boolean

Логический тип: 0-ложь, 1-истина

Данный перечень можно продолжать, однако это не имеет особого смысла. Приведенного списка типов данных вполне достаточно для решения большинства задач в плане "XML для WEB".
