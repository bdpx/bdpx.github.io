Для каждого покупателя выбрать продавца, который его обслуживает

Запрос: 
for $a in doc('file:///c:/KING/kingx/king_corp.xml')//employee
  for $b in doc('file:///c:/KING/kingx/king_corp.xml')//customers/customer
    where $a/customer/@id = $b/@id
return concat($b/name,' - ',$a/name)
Результат:

Пример демонстрирует нечто вроде "естественного соединения" - мы соединяем данные, лежащие на разных ветвях нашего дерева (это могли бы быть данные и из разных источников). Но говорить о "естественном соединении" мы можем только с кавычками, так как внешние ключи в базе данных XML не заданы.

Два оператора for создают два разных контекста.

Это может рассматриваться как вложенный цикл.
